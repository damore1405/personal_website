CCI Loan Shark

Now here's a project that I'm looking forward to finishing. At Drexel, we use an internal inventory system that was developed internally for loaning out laptops, Macbook chargers, est. Currently its running on IE and heavy on data entry. However, whoever design the labels included a QR code on all loanable hardware. Because of this, i saw an opportunity to improve it using a mobile app that would simply process the QR code of the hardware, and check it out, simplifying the process for everyone involved.

W8list

So, for some reason, some classes at Drexel don't have a waitlist. So if you dont get in before the class closes it's your job to refresh the page every day until you see a spot onen. That's stupid. Because I was too lazy to bother with this, I made my own solution which was a simple bash script that ran on the cs departments server as a cron job, and emailed me if it couldn't grep for "closed" on the class' page.

Later, I felt like changing things up after taking my systems prgramming class, and began to implement a version in C that's on track to be a threaded daemon that would watch the class without the need for a cronjob running on the server, the daemon would run instead. Wanted to have a cute way to play around with C, thrads, and HTML processing.
